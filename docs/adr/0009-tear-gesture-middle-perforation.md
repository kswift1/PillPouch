# ADR-0009: 봉지 찢기 제스처 — top edge → middle perforation 이동

## Status
Accepted — 2026-04-29

## Context

기획서(`docs/brief.md` §핵심 인터랙션 명세)는 봉지 찢기 인터랙션을 다음과 같이 정의:
- **봉지 윗부분**을 좌→우 드래그
- 0~30%: V자 컷에서 갈라지기 시작
- 30~70%: 봉지 윗부분이 따라 찢어짐
- 70~100%: **윗 조각이 살짝 들리거나 떨어짐**

이 정의 하에 [task #25](../plans/task_W2_25.md) Stage 1 시각을 구현하면서 봉지 상단 열압착 띠 바로 아래 옆구리에 V노치 2개(`sideTearNotches()`) 를 배치 — top tear 시작점 시그널.

작업지시자 시각 검토 결과 다음 이슈:

1. **옆 V노치가 `> <` 화살표로 읽힘** — "봉지 재질이 깎인 cut"이 아닌 "그려진 표시"로 인지. 노치를 진짜 cut으로 만들려면 봉지 형태 자체를 mask로 깎아야 하는데, 그래도 위치(top heat-seal 바로 아래)가 "왜 거기에?" 의문을 유발.
2. **헤더 정보와 알약 영역 사이 dotted divider가 단순 정보 구분선으로 낭비** — 그 위치는 "여기 뜯어서 약 꺼냄"의 기능적 자리.
3. **중간 perforation을 swipe gesture target으로 활용하면**:
   - 시각: 좌/우 반원 노치 + 점선 = 즉각 "여기 뜯음" 인지
   - 인터랙션: 그 라인을 따라 좌→우 swipe = 복용 완료
   - 알약은 perforation 아래에 있으니 "찢으면 약이 흘러나옴"이 직관적
   - 가설 B 강화 — 약을 꺼내는 실제 행위와 시각/인터랙션 일치

기획서 원안의 "윗부분 찢기"는 실제 한국 약국 봉지 사용 패턴과 부분 일치(상단 띠 절단)이지만, 우리 앱은 디지털 환경에서 시각·인터랙션·알약 흐름이 한 라인에 정렬되어야 학습 비용이 최소화됨. 중간 perforation은 그 정렬을 자연스럽게 만든다.

후보 비교:

| 안 | 시각 | 인터랙션 | 알약 흐름 | 가설 B |
|---|---|---|---|---|
| **원안 (top edge)** | 옆 V노치(읽기 어려움) + 상단 가상의 tear line | 봉지 윗부분 좌→우 drag | 윗 조각 떨어지면서 보임 | 강화 |
| **중간 perforation 이동** ★ | 좌/우 반원 노치 + 점선 (자명) | perforation 라인 좌→우 swipe | perforation 통해 알약 흘러나옴 | 더 강화 |
| 둘 다 (top + middle 두 인터랙션) | 복잡, 두 개의 tear 영역 | 충돌/혼란 | — | 메시지 분산 |

## Decision

**찢기 인터랙션 위치를 봉지 상단(top edge)에서 중간 perforation 라인으로 이동**.

### 새 인터랙션 명세

| 진행도 | 시각 변화 | 햅틱 |
|---|---|---|
| **0%** (대기) | 봉지 봉인 상태. 중간에 perforation(좌/우 반원 노치 + 점선) 자명. | — |
| **0~30%** | perforation 점선이 살짝 갈라지기 시작 — dash 간격 벌어짐, 좌측부터 진행 | `.soft` 진동 시작 |
| **30~70%** | perforation 따라 봉지 상하 분리 진행. 윗 부분이 살짝 들림. | 진폭 점진 증가 (5~7회 누적) |
| **70~100%** | 윗 부분이 위로 들리고, perforation 아래의 알약이 보이거나 흘러나옴 | 진폭 강함 |
| **100% 도달** | 봉지 "찢김" 상태로 고정. 윗 부분 살짝 들린 채 정착. 알약 perforation 통해 낙하 시작. | `.success` 강한 한 번 |

### 고정 유지 (원안 그대로)

- **방향**: 좌→우 (한국어 읽기 방향)
- **임계값**: 50% 미만 → spring back / 50% 이상 → auto 100%
- **햅틱**: `.soft` 누적 + `.success` 1회
- **Undo**: 5초 토스트
- **시그니처 메커니즘 의미**: "체크"가 아닌 "개봉"

### 시각 사양

- **노치 모양**: 반원 ⌒ — design system rounded 톤과 정합
- **노치 깊이**: 봉지 너비의 ~3.3% (240pt 봉지 기준 8pt). 강한 "여기 뜯음" 시그널.
- **점선 패턴**: dash `[3, 3]`, lineWidth 0.7pt, color `PPColor.textSecondary opacity 0.40`
- **위치**: 헤더 정보 영역 바로 아래 (top heat-seal 영역 끝나는 지점에서 봉지 세로의 ~30% 지점)
- **노치는 mask로 봉지 형태 자체를 깎음** — stroke만으로 그리는 것 X. 진짜 perforation 처리.

### 옆 V노치 제거

기존 `sideTearNotches()` (top heat-seal 아래 옆 V) 완전 제거. 시각 혼란 + 더 이상 기능적 의미 없음.

## Consequences

### 긍정
- 시각/인터랙션/알약 흐름이 한 라인에 정렬 → 학습 비용 최소화
- "perforation을 따라 swipe" 직관적 — 누구나 즉각 인지
- 알약 낙하(Stage 5)가 perforation 아래로 자연 낙하 — top tear의 V컷 통과보다 훨씬 자연스러움
- 옆 V노치 `> <` 시각 혼란 자연 해결
- top heat-seal은 봉지 정체성(약봉지) 시각으로만 살림 → 시각/기능 역할 분담 깔끔
- 가설 B 더 강화 — "약을 꺼내는 행위"의 시뮬레이션이 더 정확

### 부정 / 트레이드오프
- 기획서 §핵심 인터랙션 명세 변경 필요 — 본 ADR + brief.md PR
- 원안의 "봉지 윗부분 찢기" 메타포는 실제 한국 약국 봉지의 절단 패턴과 더 가까웠음 — 디지털 일관성 위해 트레이드
- 봉지 상단 부분이 "안 쓰이는 영역"이 됨 → 헤더 정보(약국명/환자/용법)로 채움으로 보완

### 위험
- middle perforation이 "약봉지"라기보다 "쿠폰 절취선"으로 인지될 가능성 — 봉지 본체의 플라스틱 + heat-seal + 슬롯 도장 시각이 약봉지 정체성을 강하게 잡고 있어 미연 방지. Stage 4 보고서에서 작업지시자 시각 재검토.
- 알약이 perforation 위(헤더 영역)에 위치 못함 → 알약 영역이 봉지 전체 높이의 ~70%로 제한. 알약 8개 이상은 좁을 수 있음. **완화**: V1 알약 ≤ 8 가정.
- 향후 Live Activity / Widget에서 작은 사이즈에서 perforation 노치가 못 보일 가능성 — Compact 형태에서는 노치 생략 + 단순 dash 라인으로 fallback (별도 디자인 task).
