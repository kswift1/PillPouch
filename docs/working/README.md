# working/ — 단계별 완료보고서 (L 태스크만)

L 태스크는 구현계획서에서 3~6단계로 쪼갠다. **각 단계 완료 후** 보고서 작성 → 작업지시자 승인 → 다음 단계.

## 파일명
`task_W{N}_{이슈}_stage{M}.md` (예: `task_W3_12_stage2.md`)

## 형식

```markdown
# task_W{N}_{이슈}_stage{M}.md

## 단계 목표 (구현계획서 §M에서 복사)

## 한 일
- 커밋 해시 + Conventional Commits 메시지
- 변경 파일 목록

## 검증 결과
- [ ] 단계 목표 달성
- [ ] 테스트 통과 (테스트 명, 수)
- [ ] (UI) 스크린샷 첨부

## 발견한 이슈
- 새로 알게 된 제약, 위험, 다음 단계 변경 필요 사항

## 다음 단계
구현계획서 §{M+1}로 진행 OK / 변경 필요 (사유)
```

## 작성 후
작업지시자 승인 ⛔ → 다음 단계 진행
