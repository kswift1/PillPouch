# plans/ — 수행계획서 + 구현계획서 (RHWP)

태스크 시작 전에 작성, **작업지시자 승인 필수**.

## 종류

| 파일 | 작성 시점 | 적용 태스크 |
|---|---|---|
| `task_W{N}_{이슈}.md` (수행계획서) | 큰 통합 시작 전 | L만 |
| `task_W{N}_{이슈}_impl.md` (구현계획서) | 코드 시작 전 | M, L |

## 구현계획서 형식 (M)

```markdown
# task_W{N}_{이슈}_impl.md

## 메타 (Issue 링크, 마일스톤, 크기, 영역, 브랜치, 예상 시간)

## 목표

## 구현 단계 (3~6단계)

## 커밋 단위 (Conventional Commits)

## 위험 요소

## 검증 (Issue 마감 조건 체크리스트)

## 다음 (이 task 완료 후)
```

## 수행계획서 형식 (L 추가)

```markdown
# task_W{N}_{이슈}.md

## 배경 / 동기

## 목표

## 범위 / 비범위

## 접근 방식 (대안 비교)

## 단계 분할 (구현계획서로 이어짐)
```

## 작성 후
1. 작업지시자에게 승인 요청
2. 승인 후에야 코드 시작
3. 무승인 진행 절대 금지
