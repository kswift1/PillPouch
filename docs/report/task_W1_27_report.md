## 부수 발견사항 (본 PR 안 변경 대상)

본 PR 진행 중 발견되어 박제에 반영된 항목들. 후속 결정 영역과 다름.

### 1. 시각 메타포 어휘 — "절취선" 통일 (Step 3-B)

코드(`PouchView.swift` / `PouchGeometry`) + ADR-0009 직접 검증 결과:

- 첨부 원문 *"봉지 윗부분을 찢어"* — 한국어 자연 해석상 인터랙션 시작 위치를 가리키는 것으로 인지됨. 실제로는 인터랙션 hit target은 perforation 라인(중간), 시각적으로 들리는 부분이 윗 절반. **부정확.**
- brief.md / ADR-0009 *"middle perforation"* — 약속된 명명이지만 실제 위치는 봉지 세로의 ~30% 지점 (top heat-seal 14 + headerCenter 38 + headerDivider 36 = 88pt / 320pt 봉지 ≈ 27.5%). **약간 부정확.**
- Identity layer는 V1 메타포에 종속되지 않게 **"봉지의 절취선을 따라"** 로 박제. §시각 메타포 첫 줄 *"한국 약봉지 띠 — 1일분 처방 봉지의 절취선 형태"* 와 어휘 통일.
- 차별점 #2도 *"실제 약봉지의 절취선을 따라 찢는 행위"* 로 통일.

### 2. (α-1) 의료 추천 vs (α-2) 인구통계 일반 권장 구분

작업지시자 발언 *"기본적인 오메가, 비타민, 마그네슘 같은건 권장 할 수 있는거 아니야?"* 계기로 의료법 경계 재분류:

- **(α-1) 개인 진단 기반 추천** ("당신의 혈액검사상 X 부족이니 X 드세요") — 의료법 §27 회색지대, 안 함
- **(α-2) 인구통계 기반 일반 권장** ("20대 남성: 오메가3·비타민D·마그네슘") — 식약처 영양소섭취기준 / 약국 카운터 안내 수준, 합법

Anti-Promise §4를 *"개인 맞춤 처방·자문 X / 인구통계 일반 권장은 정보 제공"* 로 정밀화. (α-2) 박제 위치 결정은 후속 영역 (아래).

### 1. 비즈니스 모델 결정 (광고 / 결제 / 무료 / Pro IAP / 제휴 커미션)

**배경**: 작업지시자 *"애드몹도 붙이고 하면 매일 들어오는게 좋을 것 같기도"* 발언으로 비즈니스 모델 검토 트리거. 그러나 광고 모델은 다음과 충돌 가능:

- Identity §정서 §지향 *"Things 3 + Streaks 교집합"* — 두 앱 모두 단발 결제, 광고 0
- Anti-Promise §5 *"사용자 데이터를 외부 공유하지 않는다"* — AdMob = Google 송신
- Identity §정서 §금기 *"잔소리 / 강박 / 죄책감"* — 광고 = 잔소리 인접

**결정 영역**: AdMob / 단발 결제 / 구독 / Pro 기능 IAP / 영양제 제휴 커미션 / 무료 중 선택.

**권장 시점**: V1 출시 + 30일 도그푸딩 후. 사용자 데이터 0인 상태에서 비즈니스 모델 결정은 추측 기반.

### 2. PTS / Live Activity / Remote Push 제거 검토

**배경**: 작업지시자 *"pts, 라이브액티비티 제거, remote push noti, widget은 유지"* 1차 의향 표명. 영향 범위 조사 결과 — 문서 ~600 LOC + 코드 ~400 LOC + Issue 8개 + 마일스톤 W3·W5 영향, 일정 6주 → 4.5주 단축. 백엔드는 유지 결정 (카탈로그 API + V2+ 확장 여지).

**중지 사유**: 작업지시자 *"일단 이건 나중에 고민하자"* — 본 PR 스코프 명백히 초과 + 정체성 §표면 §약속 §정체성 우선순위 §시나리오 §V1 스코프 동시 영향 = 본 PR M 사이즈 초과 + L 승격 필요.

**결정 영역**:
- A(보조) 메커니즘 재정의: Widget + Local Notification으로 충분한가?
- iOS 최소 버전: 17.2+ → 17.0+ 가능
- ADR-0001 / 0003 부분 supersede 필요
- 백엔드 책임 변경 (APNs 푸시 → 카탈로그 API)
- 마일스톤 W3 / W5 재정의

**권장 시점**: 본 PR 머지 후 즉시 또는 V1 출시 후. 별도 Issue + ADR-0011 + L 태스크 절차.
