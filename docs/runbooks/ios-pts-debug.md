# ios-pts-debug.md — iOS Push to Start 토큰 미수신 디버그

> **Status**: stub. **W5에서 채움** (PTS 토큰 발급 이슈 우회 작업 시점).

## 알려진 이슈
Apple Developer Forums에 보고된 케이스: **앱 강제 종료 후 재실행 시 PTS 토큰이 발급되지 않는 케이스** (iOS 18.x 다수 버전).
출처: "Live Activities Push-to-Start flows" 스레드 (2025년 기준 미해결).

기획서 §백엔드 도입 시 유의사항 §5에 명시.

## 채워야 할 항목

### 1. PTS 토큰 발급 정상 흐름
- [ ] 앱 시작 시 `for await tokenUpdates in Activity<...>.pushToStartTokenUpdates`
- [ ] 첫 토큰을 서버에 동기화 (`POST /v1/devices`)
- [ ] 토큰 변경 시 자동 재발송

### 2. 미수신 진단
- [ ] 앱 시작 시점에 PTS 토큰 발급 여부 체크
- [ ] N초(예: 10초) 내 토큰 안 오면 로그 + 사용자 폴백
- [ ] iOS 버전, 디바이스 모델, 앱 빌드 번호 함께 기록

### 3. 우회 / 폴백 패턴
1. **재구독**: 앱 시작 즉시 `pushToStartTokenUpdates` 다시 구독
2. **사용자 알림 UX**: "락스크린 자동 등장이 동작하지 않으면 앱을 한 번 더 열어주세요"
3. **로컬 노티 백업**: PTS 실패해도 사용자가 알림은 받도록 로컬 노티 항상 활성 (기획서 V1.0 필수)

### 4. 디버그 도구
- [ ] Xcode → Device → Console: `subsystem == "com.apple.activitykit"`
- [ ] APNs sandbox에서 토큰 등록 → 서버 로그에 토큰 확인
- [ ] 실기기 강제 종료 → 재실행 → 시간별 토큰 발급 흐름 기록

### 5. 회귀 방지 테스트
- [ ] iOS 17.2 / 17.5 / 18.0 / 18.x 별 시뮬 + 실기기 검증 매트릭스
- [ ] 앱 라이프사이클별(처음 설치, 강제 종료 후 재시작, 시스템 재부팅 후) 토큰 발급 시나리오

## 위험 메모
- 완벽한 해결책 없음을 인지 — 로컬 노티 폴백을 V1.0 필수로 유지 (기획서 §iOS 18.x 토큰 수신 이슈 우회)
- 사용자에게 "앱을 한 번 더 열어주세요" UX는 자주 띄우면 짜증 → 임계 빈도 제어
- 30일 도그푸딩(W5+) 중 자기 디바이스 동작 데이터로 패턴 확인
