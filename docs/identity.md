## 한 줄

> **영양제 트래킹/알람 앱. 단, 본질은 "먹었나 헷갈리지 않게" 만드는 것.**

---

## 3층 구조

이 앱은 세 층으로 정의된다. 외부 질문에 답할 때 **층을 분리해서 답하면 정체성이 흔들리지 않는다.**

```
┌─────────────────────────────────────────┐
│  본질 (Why)                              │
│  먹었다는 사실에 대한 신뢰감을 만든다    │
└─────────────────────────────────────────┘
                  ↑ 받쳐줌
┌─────────────────────────────────────────┐
│  표면 (What — V1)                        │
│  영양제 매일 트래킹 + 알람 + 기록        │
└─────────────────────────────────────────┘
                  ↑ 향함
┌─────────────────────────────────────────┐
│  비전 (Vision — V2+)                     │
│  영양제 컴패니언으로 확장                │
└─────────────────────────────────────────┘
```

| 외부 질문 | 답하는 층 |
|---|---|
| "이 앱 뭐야?" | **표면** — "영양제 트래킹/알람 앱" |
| "왜 만들어?" | **본질** — "먹었나 헷갈리지 않게 하려고" |
| "다음 버전엔 뭐가 추가돼?" | **비전** — "영양제 컴패니언으로 확장" |
| "다른 약 알람 앱이랑 뭐가 달라?" | **본질** — "기록 신뢰성이 본질이고, 약봉지 메타포로 풀어냄" |

> brief.md의 "메인(B) / 보조(A)" 우선순위 표는 본 문서의 본질(Why) / 표면(What) 분리와 같은 그림이다. B = 본질(기록 신뢰성), A = 표면(자동 등장 + 한 탭 찢기). A는 B를 받쳐주는 도구.

---

## 표면 (What)

V1에서 사용자가 직접 보고 만지는 것.

### 핵심 기능
- 영양제 등록 + 매일 트래킹
- 시간대별 알람 (아침 / 점심 / 저녁)
- 복용 기록 누적 (찢긴 봉지 띠 + 주간/월간 뷰)
- 락스크린 / Dynamic Island / Widget 자동 등장 (Push to Start)
- 인구통계 기반 권장 영양제 정보 (연령대·성별별 일반 가이드, Anti-Promise §4 정합 — [ADR-0011](adr/0011-recommendations-feature.md))

### 시각 메타포
- **한국 약봉지 띠** — 1일분 처방 봉지의 절취선 형태
- **가로 드래그 찢기** — 봉지의 절취선을 따라 찢어 약을 먹는 행위 시뮬레이션

### 카테고리
영양제/약 트래킹 카테고리 안에서 **"신뢰성"** 으로 차별화.
필리, MediSafe, 약 알람류와 같은 카테고리이되 본질이 다름.

**카테고리 떠나기 X — 카테고리 안에서 본질을 다르게 정의.**

---

## 본질 (Why) — 가장 중요한 층

이 층이 흔들리면 모든 디자인 결정이 mess가 된다. 시안/기능 평가 시 **반드시 이 층으로 돌아와 검증.**

### 진짜 페인포인트
*"매일 영양제 챙기는데, 먹었나 안 먹었나 헷갈린다."*

이건 까먹는 문제가 아니라 **기억의 불확실성** 문제다. 영양제는 챙기려고 사 놓고도, 먹은 사실이 신뢰할 수 있는 기록으로 남지 않으면:
- 두 번 먹거나 (과다)
- 안 먹거나 (누락)
- 매일 *"나 먹었나?"* 하는 작은 인지 부담이 누적

→ 결국 영양제 챙김 자체가 무너진다.

### Pill Pouch의 답
**"먹었다는 사실에 대한 신뢰감"** 을 만든다.

명확한 시각적 증거(찢긴 약봉지)와 누적 기록을 통해 *"내가 먹었는가?"* 라는 의문이 생기는 순간 즉시 해소되도록.

### 핵심 가설
> 영양제 챙기는 사람들의 진짜 페인포인트는 '헷갈리는 인지 부담'이다. 명확한 시각적 증거와 누적 기록을 보여주면, 그 부담이 사라지고 영양제 챙김 자체가 더 지속된다.

> *(brief.md §핵심 가설은 동일 가설을 V1 메타포(찢긴 약봉지)에 결합한 형태로 진술한다. 두 표현은 같은 가설.)*

## 비전 (Vision) — V2+

> 이 섹션은 **V1 정체성과 분리**되어 있다. V1 스코프 결정 시 이 섹션을 근거로 들지 않는다. 외부 공유용으로만 참조.

### 장기 방향
영양제 트래킹의 신뢰성 기반 위에, **영양제 컴패니언**으로 확장.

### 가능한 확장 영역 (V2+)
- 영양제 조합 안전성 (성분 중복·상호작용 경고 — AI 또는 룰 기반, 구현 방식 미정)
- 처방약·영양제 통합 (한 곳에서 트래킹)

> 가족 공유 / HealthKit 통합 / 컨디션 상관관계는 V2+에서도 다루지 않는다 ([brief.md §Non-goals](brief.md#non-goals-v1에서-의도적으로-안-만드는-것)와 일치).

### 비전과 V1의 관계
비전은 V1의 결과로 정당화된다. V1에서 가설(신뢰성)이 검증되어야 V2 확장의 토대가 생긴다.
**검증 없는 확장 금지.** (검증 기준: [brief.md §검증 가능한 명제](brief.md#검증-가능한-명제))

## 차별점 3가지

다른 영양제/약 알람 앱과 Pill Pouch를 구분짓는 것.

### 1. 한국 약봉지 띠 메타포
- 글로벌 영양제 앱 어디에도 없는 시각
- App Store 첫 스크린샷에서 즉시 차별
- 한국 사용자에게 정서적 친숙

### 2. 가로 드래그 찢기 인터랙션
- 체크박스가 아니라 **개봉**
- 실제 약봉지의 절취선을 따라 찢는 행위 = 약 먹는 행위 = 앱에서의 기록
- 메타포와 인터랙션이 한 몸

### 3. 누적 기록의 비중
- 오늘만 보여주는 앱 X
- 여러 시간 단위(하루·주·월)로 신뢰성 있는 기록 시각화
- "기록 신뢰성"이 메인 화면에서 즉시 느껴짐

## 약속 (Promise)

사용자에게 보장하는 것.

- **헷갈리지 않게 한다** — 먹었나 안 먹었나 의문이 생기는 순간 즉시 해소
- **매일 자동으로 락스크린에 등장한다** — 앱 안 열어도 (Push to Start)
- **기록은 절대 사라지지 않는다** — 찢긴 봉지는 영구 보존
- **잔소리하지 않는다** — 강요/강박 없는 차분한 동반

---

## 안 하는 약속 (Anti-Promise)

명시적으로 **하지 않을 것.** 외부에서 요청 들어와도 거절할 영역.

- **영양제를 처방하지 않는다** — 의료 행위 아님
- **진단/자문하지 않는다** — 의료법 회색지대 회피
- **알람을 강요하지 않는다** — A는 B를 위한 도구지 본질 아님
- **개인 맞춤 처방·자문을 하지 않는다** — 인구통계 일반 권장(연령대·성별별 표준 영양제 가이드)은 정보 제공이지 의료 자문이 아니다
- **사용자 데이터를 외부 공유하지 않는다** — 영양제 복용 기록은 민감

## 변경 정책

이 문서는 **거의 변경되지 않는다.** SoT 2층 구조([ADR-0010](adr/0010-sot-identity-brief-two-layer.md))의 상위 layer로, 변경 시 영향 범위가 표면 layer(brief.md) 전체에 미친다.

변경 시:

1. **ADR 선행** — 본질(Why) 변경은 ADR + 작업지시자 직접 결정만
2. 이전 버전을 git으로 보존
3. 변경 사유를 §변경 로그에 기록
4. **brief.md 양방향 점검** — 정체성 변경은 표면 기능에 영향, brief.md cross-link 정합 확인

다음과 같은 변경은 **신중히** 검토:
- 본질(Why) 변경 → 앱의 다른 앱이 됨, 사실상 새 프로젝트
- 차별점 변경 → 카테고리 안에서의 위치 변경
- 비전 변경 → 장기 방향 전환
